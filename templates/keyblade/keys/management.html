This page lists keys authorized to perform actions on your behalf.

A key is used for applications to perform actions as if you were logged in.

Keys must be properly secured. An insecurely stored key can take any action as your user.

Treat keys like any other API token or password. Do not share them or check them into code repositories.

You can use keys in the C3s API by setting the Authorization header. For example: Authorization: Bearer c3s-0-a1b2c3d4e5f6...

Keys work for the C3s API as well as the object data retrieval URLs. They will not work for text indices. The API must be used for indices.
